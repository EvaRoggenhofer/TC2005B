¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Java y Javascript no tienen mucho en común. Javascript es un lenguaje de programación basado en texto, no requiere compilación. Java es un lenguaje compilado. Ambos lenguajes utilizan una sintaxis estilo C, pero existen algunas diferencias.
Java utiliza una sintaxis de lenguaje de programación orientada a objetos, creada mediante clases y objetos.
JavaScript utiliza una sintaxis de programación basada en texto al estilo de C, utiliza convenciones de nomenclatura similares a las de Java.

¿Qué métodos tienen los arreglos? (Menciona al menos 5*)

.length devuelve la longitud (tamaño) de un array
push() añade un nuevo elemento a un array (al final) y devuelve la nueva longitud del array
toString() convierte un arreglo en un string de los valores del arreglo (separados por comas)
pop() elimina el último elemento de un array y devuelve el valor que se eliminó
concat() crea un nuevo arreglo fusionando (concatenando) arreglos existentes

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Cualquier cambio en sus valores se aplica a todo el código. Un cambio de este tipo suele hacerse por error.
El código se vuelve menos flexible cuando utilizamos variables globales. Si dos módulos comparten una variable global,
no podemos modificar uno sin tener en cuenta cómo afecta al otro.
