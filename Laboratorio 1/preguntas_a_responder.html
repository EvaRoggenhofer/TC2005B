¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET se utiliza para solicitar datos de un recurso especificado del sevidor. lo usa cuando buscamos algo en el navegador. GET: solo tiene head, no body POST se utiliza para enviar datos a un servidor para crear/actualizar un recurso. POST si tiene body (= cifrado en https). formas: con POST. HEAD (Lalo: kein Kommentar dazu) es casi idéntico a GET, pero sin el cuerpo de la respuesta. Si GET /users devuelve una lista de usuarios, entonces HEAD /users hará la misma petición pero no devolverá la lista de usuarios. Las peticiones HEAD son útiles para comprobar lo que devolverá una petición GET antes de hacer realmente una petición GET - como antes de descargar un archivo grande o un cuerpo de respuesta. PUT se utiliza para enviar datos a un servidor para crear/actualizar un recurso. La diferencia entre POST y PUT es que las peticiones PUT son idempotentes. Es decir, llamar a la misma solicitud PUT varias veces siempre producirá el mismo resultado. Por el contrario, llamar a una petición POST repetidamente tiene efectos secundarios de crear el mismo recurso varias veces. El método PATCH se utiliza para aplicar modificaciones parciales a un recurso. El método DELETE elimina el recurso especificado.
